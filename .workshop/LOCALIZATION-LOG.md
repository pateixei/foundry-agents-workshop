## Decisões de Terminologia

Consulte [GLOSSARY.md](GLOSSARY.md) para o glossário completo EN → pt-BR.

### Decisões-chave tomadas:

1. **Nomes de serviço Azure** mantidos em inglês em todas as traduções (Azure AI Foundry, Azure Container Apps, etc.)
2. **Capability Host** mantido em inglês — é um nome próprio do recurso no Azure
3. **Contêiner** usado como tradução de "container" (forma brasileira com acento)
4. **Deploy** mantido em contexto CLI, "implantação" usado em texto corrido
5. **Hosted Agent / Agente Hospedado** — padrão de primeira ocorrência com ambos os termos
6. **Workshop** mantido em inglês (padrão da indústria no Brasil)
7. **Lab** mantido em títulos, "laboratório" usado em texto corrido

## Desafios e Resoluções

| Desafio | Resolução |
|---------|-----------|
| Termos sem tradução consolidada em pt-BR (ex.: "grounding", "function calling") | Mantidos em inglês com nota no glossário |
| Arquivos com muito código embutido (lessons 5-8) | Tradução cuidadosa do texto ao redor, preservando blocos de código intactos |
| Tabelas com mistura de texto e código | Células de código mantidas em inglês, células descritivas traduzidas |
| Arquivos com instruções multiplataforma (prereq, setup-guide) | Traduzidos mantendo labels de OS em inglês (PowerShell/Bash) e todos os blocos de código intactos |
| Labs e demos com heavy code blocks | Traduziu-se toda a prosa instrucional mantendo blocos de código byte-idênticos |
